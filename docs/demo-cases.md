# 데모 케이스

예선 제출용 3분 데모는 6개 tool이 한 번씩 호출되는 단일 흐름으로 구성한다.
성공 흐름 1개 + 분기(재질문) 2개로 "AI가 모르면 추정하지 않고 되묻는다"는
안전성을 함께 보여준다.

## 성공 케이스: 병원 예약 → 가족 공유 → 약 등록 → 조회 → 복약 완료 → 공유 해제

### 1단계: 병원 예약 등록

```text
사용자: 내일 오전 10시 분당서울대병원 예약 있어
```

호출: `add_hospital_appointment`
기대: `resolved_datetime` KST 기준 내일 10:00, `confirm_message` 반환

### 2단계: 가족 공유 켜기 + 공유문 생성

```text
사용자: 딸한테도 알려줘
```

호출: `update_share_scope(enable)` → 이어서 `generate_share_summary`
기대 응답에 아래 취지 포함:

```text
딸에게 직접 보내려면 가족 인증이 필요해요. 우선 이 문구를 복사해서 보내실 수 있어요.
```

### 3단계: 약 등록

```text
사용자: 혈압약 아침저녁 식후로 챙겨줘
```

호출: `add_medication_text`
기대: 아침/저녁 슬롯 등록 확인

### 4단계: 오늘 일정 조회

```text
사용자: 오늘 뭐 있어?
```

호출: `list_today_schedule`
기대: 병원 예약 + 복약 상태(아침=미확인) 요약 반환

### 5단계: 복약 완료 처리

```text
사용자: 아침약 먹었어
```

호출: `mark_medication_taken` (medication_id/name 없이 발화)
기대: 아침 슬롯 약이 1건이라 자동 매칭 → 완료 처리

### 6단계: 가족 공유 해제

```text
사용자: 딸 공유는 꺼줘
```

호출: `update_share_scope(disable)`
기대: 공유 비활성화 확인

## 분기 케이스 1: 날짜 모호

```text
사용자: 다음주 화요일 병원 예약 등록해줘
```

기대: `hospital_name` 없이는 등록 안 됨 → `need_clarification`("병원 이름을 말씀해주시겠어요?" 류)
또는 날짜 자체가 모호하면 `reason: "ambiguous_date"`로 재질문. 추정해서 임의 등록하지 않는다.

## 분기 케이스 2: 같은 시간대 약 2건 이상

```text
사전 등록: 아침 혈압약, 아침 당뇨약
사용자: 아침약 먹었어
```

기대: `mark_medication_taken`이 `status: "need_clarification"`,
`reason: "multiple_medications"` 반환 — "아침약이 두 개 등록되어 있어요. 어떤 약인지
말씀해주시겠어요?"

## 데모 운영 원칙

- 능동 알림(30분 재확인)은 이번 데모에 포함하지 않는다 — 조회형(`list_today_schedule`)만으로
  핵심 가치를 보여준다.
- 가족 공유는 "직접 발송"이 아니라 "공유문 생성 + 직접 전달 안내"로만 시연한다.
- 이미지/OCR 약 등록은 이번 데모에 포함하지 않는다 — 텍스트 등록만 시연한다.

# 챙김비서 작업 지침

이 프로젝트는 `dukpin` 레포와 완전히 분리된 별도 프로젝트입니다.
덕핀 운영지침(관리자/사용자 API 원칙, 인코딩 기준 등)은 이 프로젝트에 적용하지 않습니다.

## 프로젝트 개요

- 이름: 챙김비서
- 목적: Kakao AGENTIC PLAYER 10 예선 제출용 MCP 서버
- 핵심 흐름: 안내문(사진/OCR 텍스트) 추출 → 체크리스트 생성 → 가족방 공유문 생성

## MVP 범위 고정

포함:
- 학교 가정통신문 / 체험학습 안내문 / 준비물 공지 / 학급 행사 안내
- 날짜·장소·준비물·제출물·마감일 추출
- `confidence_level`(high/medium/low), `evidence_type`(explicit/inferred/missing), `needs_review`, `review_reason` 포함
- 준비/제출물 체크리스트 생성
- 가족방 공유용 메시지 생성

제외 (MVP 및 본선 확장 모두):
- 학원 숙제, 교재 페이지 범위, 과목별 진도 관리
- 부모님 병원/약 복용 관리
- 위치추적
- 결제/정산
- 자동 예약
- 아이/보호자 역할 자동 분담 (v2 이후 검토)
- 실제 알림 발송, 실제 톡캘린더 연동 (MVP는 payload/기준만 정의)

본선 확장 후보 (예선 자료에는 계획으로만 명시, 과대 약속 금지):
1. 완료 체크 상태 관리
2. 미완료 항목 리마인드
3. 전날/당일 준비 브리핑

## 데모 안정성 원칙

- "어떤 사진이든 인식 가능"을 주장하지 않는다.
- 데모는 사전 검증된 안내문 3종(체험학습/미술준비물/학급발표회) + 실패 방어 2종(OCR 깨짐/날짜 없음)으로 고정한다.
- 사진 업로드는 데모 핵심으로 유지하되, 사전 검증된 OCR 텍스트 fallback을 둔다.
- `confidence` 연속값(0~1 숫자)을 사용하지 않는다. `confidence_level`과 `evidence_type`만 사용한다.

## 알림/마감 기준 (고정값)

| 항목 | 기준 |
|---|---|
| 오전 행사 | `event_time < 12:00` |
| 오후 행사 | `12:00 <= event_time < 18:00` |
| 저녁 행사 | `event_time >= 18:00` |
| 전날 준비물 체크 | 전날 21:00 |
| 당일 아침 체크 | 당일 07:30 |
| 제출물 마감 전날 | 21:00 |
| 제출물 마감 당일 | 07:30 |
| 행사 시간 없음 | 전날 21:00만 후보 생성 |
| 날짜 불명확 | 알림 생성하지 않고 `needs_review=true` |

## 작업 원칙

- 결정은 사용자가 한다. Codex는 의견·트레이드오프를 제시하되 결정을 대신하지 않는다.
- git 커밋은 사용자가 명시적으로 요청할 때만 한다.
- 범위를 벗어나는 기능 추가(학원 숙제 등) 요청이 들어오면 먼저 트레이드오프를 보고하고 승인 후 진행한다.
